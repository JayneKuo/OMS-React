# PO Detail Page Single View Design

## 设计理念

基于用户反馈"用户还要点击很多下才可以看到相关信息更不好了"，重新设计为单一视图布局，让用户无需点击切换就能一次性看到所有关键信息。

## 核心原则

### 1. 零点击信息获取
- **一屏展示**: 所有关键信息在一个页面内展示
- **无需切换**: 取消标签页设计，避免用户点击切换
- **信息完整**: 包含所有必要的业务信息

### 2. 视觉层次优化
- **重要信息突出**: PO编号、订单金额、供应商名称等关键信息突出显示
- **逻辑分组**: 相关信息合理分组，便于快速扫描
- **色彩区分**: 使用不同颜色区分不同类型的信息

### 3. 空间效率最大化
- **紧凑布局**: 在保证可读性的前提下最大化信息密度
- **响应式设计**: 适配不同屏幕尺寸
- **合理间距**: 保持视觉舒适度

## 新布局结构

### 第一行：核心业务信息 (3列布局)
```
┌─────────────────┬─────────────────┬─────────────────┐
│   订单信息      │   供应商信息    │  金额与进度     │
│                 │                 │                 │
│ • PO编号(突出)  │ • 供应商名称    │ • 订单金额(突出)│
│ • 原始PO/参考号 │ • 联系人信息    │ • 发货进度条    │
│ • 数据来源      │ • 电话/邮件按钮 │ • 收货进度条    │
│ • 创建时间      │ • 供应商地址    │ • 付款条款      │
│ • 预计到货      │                 │ • 交货条款      │
│ • 关联PR        │                 │                 │
└─────────────────┴─────────────────┴─────────────────┘
```

### 第二行：地址与物流信息 (2列布局)
```
┌─────────────────────────────┬─────────────────────────────┐
│        地址信息             │        物流跟踪             │
│                             │                             │
│ • 发货地址 (供应商)         │ • 发货记录                  │
│   - 供应商名称              │   - ASN编号/运单号          │
│   - 详细地址                │   - 承运商/数量             │
│                             │   - 发货时间/预计到达       │
│ • 收货地址 (仓库)           │                             │
│   - 仓库名称                │ • 收货记录                  │
│   - 详细地址                │   - 收货单号/数量           │
│   - 仓库编码                │   - 收货时间/收货人         │
│                             │   - 备注信息                │
└─────────────────────────────┴─────────────────────────────┘
```

## 关键改进点

### ✅ 解决用户痛点
1. **消除点击**: 取消标签页，所有信息一屏展示
2. **提高效率**: 用户可以快速扫描所有信息
3. **减少认知负荷**: 不需要记住信息在哪个标签页

### ✅ 信息完整性
1. **发货地址**: 完整显示供应商地址
2. **收货地址**: 包含仓库名称、地址、编码
3. **物流记录**: 发货和收货记录的完整信息
4. **商务条款**: 付款和交货条款直接显示

### ✅ 视觉优化
1. **突出显示**: PO编号、订单金额、供应商名称使用彩色背景突出
2. **进度可视化**: 发货和收货进度使用进度条直观显示
3. **操作便捷**: 电话和邮件按钮直接可点击
4. **状态清晰**: 使用不同颜色区分发货和收货信息

## 响应式设计

### 大屏幕 (≥1024px)
- 第一行：3列布局
- 第二行：2列布局
- 信息密度最大化

### 中等屏幕 (768px-1023px)
- 第一行：2列布局，第三列换行
- 第二行：2列布局
- 保持信息可读性

### 小屏幕 (<768px)
- 所有卡片单列堆叠
- 保持信息层次
- 优化触控操作

## 技术实现特点

### 1. 性能优化
- **单一渲染**: 避免标签页切换的重渲染
- **数据预加载**: 所有数据一次性加载
- **组件复用**: 统一的卡片和布局组件

### 2. 可维护性
- **模块化设计**: 每个信息块独立组件
- **统一样式**: 一致的设计语言
- **类型安全**: TypeScript类型检查

### 3. 可访问性
- **键盘导航**: 支持Tab键导航
- **屏幕阅读器**: 语义化HTML结构
- **对比度**: 符合WCAG标准

## 数据展示优化

### 关键信息突出
- **PO编号**: 蓝色背景，大字体显示
- **订单金额**: 橙色背景，突出显示
- **供应商名称**: 绿色背景，便于识别

### 进度可视化
- **发货进度**: 蓝色进度条，显示百分比和具体数量
- **收货进度**: 绿色进度条，显示百分比和具体数量

### 操作便捷性
- **联系按钮**: 电话和邮件按钮直接可点击
- **复制功能**: PO编号一键复制到剪贴板
- **关联导航**: PR编号可点击跳转
