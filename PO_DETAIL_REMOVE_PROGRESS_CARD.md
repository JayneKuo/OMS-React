# PO Detail Page - Remove Progress Card

## 修改内容

### 删除的组件
- **金额与进度卡片** - 完全移除了包含订单金额、发货进度、收货进度的第三个卡片

### 布局调整
- **从3列改为2列布局** - `grid-cols-1 lg:grid-cols-3` 改为 `grid-cols-1 lg:grid-cols-2`
- **保留核心信息** - 订单信息和供应商信息卡片保持不变

### 信息整合
- **付款条款和交货条款** - 从删除的进度卡片移动到订单信息卡片中
- **保持信息完整性** - 重要的商务条款信息不会丢失

### 代码清理
- **移除Progress组件导入** - `import { Progress } from "@/components/ui/progress"`
- **移除DollarSign图标导入** - 从lucide-react导入中移除`DollarSign`

## 优势

### 1. 简化布局
- **减少视觉复杂度** - 从3个卡片减少到2个
- **更好的空间利用** - 2列布局在大多数屏幕上更合适
- **信息更集中** - 相关信息归类更合理

### 2. 保持功能完整
- **商务条款保留** - 付款和交货条款移到订单信息中
- **核心信息不丢失** - 所有重要信息都有合适的位置
- **逻辑分组** - 订单相关信息集中在一个卡片中

### 3. 性能优化
- **减少组件** - 移除了Progress组件和相关计算
- **简化渲染** - 更少的DOM元素
- **代码清理** - 移除了不必要的导入

## 影响分析

### 移除的功能
- **进度条可视化** - 发货和收货进度的可视化展示
- **订单金额突出显示** - 大字体的金额展示
- **进度百分比** - 数值和百分比的进度显示

### 替代方案
- **商品明细表格** - 进度信息在商品明细的汇总区域仍然可见
- **状态徽章** - 通过状态徽章了解整体进度
- **汇总信息** - 在商品明细下方的汇总区域查看金额

## 用户体验

### 正面影响
- **页面更简洁** - 减少信息过载
- **布局更平衡** - 2列布局视觉更协调
- **加载更快** - 减少组件渲染时间

### 需要注意
- **进度信息** - 用户需要在商品明细区域查看详细进度
- **金额信息** - 订单总金额在商品明细汇总区域显示
