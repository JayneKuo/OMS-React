# AI 新窗口启动提示词

以下内容可直接复制到任何新窗口，作为该窗口的启动指令。

---

## 标准启动提示词（通用版）

先阅读并严格遵守以下两个文件：

- [01-Stripe后台UI统一规范.md](01-Stripe后台UI统一规范.md)
- [02-Stripe后台技术实现规范.md](02-Stripe后台技术实现规范.md)

要求如下：

1. **UI 层必须严格服从《01-Stripe后台UI统一规范》**
   - 不允许自由发挥视觉风格
   - 不允许重造页面 grammar
   - 不允许使用营销型 SaaS 风格
   - 不允许偏离 Stripe 风格后台的布局、组件、交互和字段展示规则

2. **技术实现必须严格服从《02-Stripe后台技术实现规范》**
   - 使用 TypeScript、React、Next.js、Tailwind、shadcn/ui、TanStack Table、React Hook Form + Zod 的既定实现方式
   - 组件分层、文件夹结构、模板抽象、字段映射、状态映射必须按规范组织
   - 不允许在页面里直接堆叠大段未抽象 JSX 去绕开系统

3. **如果要搭建系统，请按这个顺序执行**
   - 先搭 shell
   - 再搭 page archetype
   - 再搭 shared components
   - 再搭 domain sections
   - 最后接业务数据与字段映射

4. **如果要做具体页面，请自动套用既有页面 archetype**
   - list
   - detail
   - create/edit
   - settings hub
   - reports hub
   - approval/process
   - developer/workbench

5. **业务定制只允许改这些内容**
   - 字段
   - 数据
   - 筛选器
   - 动作
   - tabs
   - section 内容
   - 文案中的业务术语

6. **未经明确要求，不允许改这些内容**
   - shell 布局
   - UI 风格
   - 组件层级
   - 页面模板体系
   - token 体系
   - 按钮主次关系
   - 列表/详情/设置/报表的结构语法

7. **开始实现前，你必须先判断并说明**
   - 当前任务属于哪种页面 archetype
   - 主动作是什么
   - 次动作是什么
   - 关键字段来自哪个对象展示模板
   - 页面里哪些区域属于 main content，哪些属于 side metadata/config

如果这些规则与当前实现冲突，优先保持这两个规范文件中的系统一致性。

## 标准启动提示词（从系统架构开始）

先阅读并严格遵守：
- [01-Stripe后台UI统一规范.md](01-Stripe后台UI统一规范.md)
- [02-Stripe后台技术实现规范.md](02-Stripe后台技术实现规范.md)

现在请从“系统架构、导航、页面容器、基础组件层”开始搭建，不要直接堆业务页面。

执行顺序必须是：
1. shell
2. page archetype
3. shared components
4. domain sections
5. data mapping

不要自由发挥风格，不要引入新的 UI 系统，不要偏离 Stripe 风格后台规范。

## 标准启动提示词（做具体页面）

先阅读并严格遵守：
- [01-Stripe后台UI统一规范.md](01-Stripe后台UI统一规范.md)
- [02-Stripe后台技术实现规范.md](02-Stripe后台技术实现规范.md)

现在请生成一个符合这套系统的页面。

要求：
- 自动选择正确页面 archetype
- 自动选择正确字段展示层级
- 自动遵守按钮层级和 toolbar 规则
- 只按业务需求修改字段、数据、动作和筛选项
- 不允许偏离 Stripe 风格后台的布局和交互体系

输出前请先说明：
- 你选择了哪种 archetype
- 你使用了哪些关键字段
- 主动作和次动作分别是什么
