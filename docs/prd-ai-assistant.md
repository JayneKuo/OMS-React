# 产品需求文档：OMS AI 助手

## 1. 背景

OMS 系统功能模块多（订单、采购、库存、物流、退货、客户、自动化等），运营人员日常工作需要在多个页面间频繁切换。新员工上手周期长，老员工也经常需要多步操作才能完成一个简单任务（比如查一个订单的异常原因，需要先进订单列表、找到订单、进详情、看异常标签、再去事件管理看日志）。

同时，业务数据散落在各个模块中，运营主管想看一个跨模块的数据汇总（比如"本月异常率和上月对比"）往往需要自己导出数据手动分析。

Shopify 在 2024-2026 年推出并持续迭代的 Sidekick AI 助手验证了一个方向：在后台管理系统中嵌入一个自然语言交互的 AI 助手，可以显著降低操作门槛、提升运营效率。

我们希望在 OMS 中引入类似的 AI 助手能力。

## 2. 目标

### 业务目标

- 降低运营人员的操作成本，减少完成常见任务所需的步骤和时间
- 降低新员工的上手门槛，通过 AI 引导快速熟悉系统
- 提升异常发现和处理的时效性，通过主动洞察让问题更早被发现

### 衡量指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 任务效率提升 | 常见查询任务耗时降低 50% | 对比有/无 AI 助手时完成同一任务的平均时间 |
| 用户采纳率 | 上线 1 个月后 >= 30% 日活用户使用 | 埋点统计面板打开次数 / DAU |
| 对话深度 | 平均 >= 3 轮/次 | 统计每次对话的消息轮次 |
| 新人上手周期 | 缩短 30% | 对比新员工独立完成首个任务的时间 |

## 3. 用户角色与场景

### 3.1 用户角色

| 角色 | 日常工作 | 痛点 |
|------|----------|------|
| 运营专员 | 处理订单异常、退货、发货跟踪 | 异常处理流程长，需要跨多个页面操作；重复性查询多 |
| 运营主管 | 监控业务指标、审批、团队管理 | 数据散落各模块，缺少跨模块汇总视角；异常发现滞后 |
| 新入职员工 | 学习系统、处理简单任务 | 不知道功能在哪、不清楚操作流程、怕操作错误 |
| 商户管理员 | 管理产品、库存、定价 | 产品和库存操作步骤多，批量操作不直观 |

### 3.2 核心场景

**场景 A：快速查数据**
运营主管每天早上想看昨天的业务概况。现在需要打开仪表板、切换时间范围、逐个看各项指标。希望能直接问一句"昨天的订单量和异常率是多少"就拿到答案。

**场景 B：异常处理**
运营专员收到异常通知，需要查原因。现在要进订单详情、看异常标签、去事件管理查日志、判断根因、决定处理方式。希望能输入订单号，AI 直接告诉根因和建议操作。

**场景 C：找功能入口**
新员工想处理一个退货，但不知道退货管理在哪。希望能问 AI"退货管理在哪"，直接给一个链接点过去。

**场景 D：主动发现问题**
运营主管没有主动去看数据，但今天的异常率其实比昨天高了很多。希望 AI 能主动提醒"今天异常率上升了 23%，要不要看看？"

**场景 E：操作指引**
商户管理员想创建一个新产品，但不确定流程。希望能问 AI"怎么创建产品"，AI 给出步骤说明和对应页面的链接。

**场景 F：自动化需求**
运营主管想设置一个规则：库存低于安全水位时自动通知采购。希望能用自然语言描述需求，AI 给出配置指引。

**场景 G：智能分仓查询**
运营人员需要了解某个订单的分仓结果、分仓规则命中情况，或者评估是否需要在某个区域开设新仓库。希望能直接问 AI"查询订单 SO12345 的分仓结果"，AI 给出分仓详情和分析建议。

**场景 H：装箱计算**
仓配人员需要知道一票货怎么装箱最优、用什么箱型、计费重多少。希望能描述商品信息和箱型，AI 直接给出精确的装箱方案。

## 4. 功能描述

### 4.1 AI 助手入口

用户在系统的任意页面都能看到 AI 助手的入口——顶部导航栏右侧的 Sparkles 图标按钮。点击后页面右侧平滑展开 AI 助手面板（宽度 400px，动画 300ms ease-in-out），主内容区自动收窄，不产生水平滚动条。再次点击按钮或面板内的关闭按钮可收起面板。按 Escape 键也可关闭面板。

面板展开时，AI 按钮显示激活状态。面板状态在页面切换时保持。

面板头部显示"OMS AI 助手"标题和"Beta"标签，副标题为"智能问答 / 操作执行 / 主动洞察"。当有活跃对话时，头部切换为显示对话标题和消息数量。

### 4.2 欢迎视图与快捷建议

打开 AI 助手且无活跃对话时，展示欢迎视图：

- 顶部为 Sparkles 图标 + "有什么可以帮你？"标题 + 简短能力描述
- 下方按能力域分组展示快捷建议（2 列网格布局），每项包含图标和标签
- 默认展示前 4 个分类，底部提供"展开更多"按钮，点击展开剩余分类，再次点击收起
- 点击任一快捷建议，自动创建新对话并发送对应的预设提问

### 4.3 AI 能覆盖的能力域

| 能力域 | 用户能问什么 | AI 能做什么 | 快捷建议 |
|--------|-------------|------------|----------|
| 数据与分析 | "本月销售额多少""和上月对比怎么样""帮我导出订单数据" | 返回数据摘要，提供报表页面链接 | 查看销售数据、对比业绩表现、导出数据报表 |
| 产品与库存 | "怎么创建产品""哪些商品快缺货了""怎么管理产品集合" | 操作步骤指引，提供对应页面链接 | 创建/编辑产品、查询库存水平、管理产品集合 |
| 智能分仓 | "查询订单分仓结果""分析是否需要开仓""分仓规则怎么配置" | 分仓查询结果、开仓可行性分析、规则配置指引 | 分仓查询、开仓分析、分仓规则 |
| 客户管理 | "帮我找高价值客户""某个客户的订单历史" | 查询指引，提供客户管理页面链接 | 创建客户细分、查找客户信息 |
| 定价与促销 | "有什么定价策略建议""最近促销效果怎么样" | 策略建议和分析指引 | 定价策略建议、分析促销效果 |
| 自动化 | "帮我创建一个低库存通知规则""有什么推荐的自动化" | 配置指引，提供自动化页面链接 | 创建工作流、自动化建议 |
| 导航与引导 | "退货管理在哪""怎么配置店铺设置""最近有什么新功能" | 直接给出页面链接，操作指引 | 快速导航、设置指导、发布历史 |
| 订单与异常 | "SO12345 什么状态""今天有多少异常订单" | 订单全景查询、异常诊断指引 | 查询订单、异常分析 |
| 装箱与物流 | "这票货怎么装箱""计费重多少""用什么箱型" | 精确装箱计算、箱型推荐、计费重计算 | 装箱计算、箱型选择 |
| 集成中心 | "Shopify 连接器什么状态""最近同步有没有失败" | 连接器状态查询、同步记录、健康检查 | 连接器状态、同步记录 |

此外，AI 还能识别以下场景并给出对应回复：

- 数据导出：识别导出/下载/CSV/Excel 等关键词，给出导出操作步骤
- 系统设置：识别设置/配置相关问题，给出店铺设置指引
- 新功能更新：识别"新功能""更新"等关键词，展示最近的功能更新列表

当输入无法匹配任何能力域时，返回通用引导回复，列出所有能力域供用户选择。

### 4.4 主动洞察（Pulse）

AI 不只是被动回答问题，还能主动分析业务数据，发现值得关注的变化并推送给用户。欢迎视图顶部（快捷建议上方）展示 Pulse 洞察卡片区域，标题为"智能洞察"。

每张洞察卡片包含：

- 语义色彩图标（橙色/绿色/蓝色）
- 标题和描述
- 可点击的操作按钮（点击后创建新对话并发送对应查询）
- 关闭按钮（hover 时显示，关闭后当前会话不再显示）

第一期预设 3 张洞察卡片：

| 洞察 | 颜色 | 标题 | 描述 | 操作 |
|------|------|------|------|------|
| 异常率变化 | 橙色 | 异常率上升 | 今日订单异常率较昨日上升 23% | 查看详情 |
| 销售趋势 | 绿色 | 销售增长 | 本周销售额环比增长 12% | 查看报表 |
| 库存预警 | 蓝色 | 库存预警 | 3 个 SKU 库存低于安全水位 | 查看库存 |

### 4.5 对话交互

#### 输入区

- 底部固定的自动伸缩 Textarea（最小 36px，最大 100px）+ 发送按钮
- Enter 发送消息，Shift+Enter 换行，底部显示快捷键提示
- 面板展开时输入框自动获取焦点（延迟 300ms，等待动画完成）
- 发送按钮在输入为空或 AI 正在回复时禁用，加载时显示旋转动画

#### 消息展示

- 用户消息：右对齐，紫色背景，圆角气泡，带 User 头像
- AI 消息：左对齐，灰色背景，圆角气泡，带 Bot 头像
- 每条消息显示发送时间（时:分格式）
- AI 正在回复时显示加载状态：Bot 头像 + "思考中..." + 旋转动画
- 新消息到达时自动滚动到底部

#### 结构化回复

AI 回复支持以下内容块类型：

- 纯文本段落：支持换行和列表格式
- 导航链接卡片：紫色边框 + 图标 + 文本 + 箭头，点击跳转到 OMS 页面
- 内嵌图表：当 AI 回复包含数据分析结果时，在对话气泡内渲染可视化图表（详见 4.9）

消息内容使用 `[文本](路径)` 语法存储导航链接，渲染时解析为链接卡片。
图表数据使用 `:::chart{type="bar"} ... :::` 语法存储，渲染时解析为对应图表组件。

#### AI 消息工具栏

每条 AI 消息底部提供：

- 复制按钮：点击复制消息纯文本内容（去除链接和图表语法），复制成功后图标变为绿色对勾，1.5 秒后恢复
- 反馈按钮：有帮助 / 没帮助，点击切换选中状态，再次点击取消

### 4.6 多对话管理

- 面板头部提供"新建对话"按钮，点击创建空白对话
- 面板头部提供"回到首页"按钮（仅在有活跃对话时显示），点击返回欢迎视图
- 用户在无活跃对话时发送消息，自动创建新对话
- 每个对话拥有独立的消息列表
- 对话标题从首条用户消息提取（前 30 字符，超出显示"..."）
- 对话上限 50 条，超出时自动移除最早的对话

### 4.7 对话历史

- 面板头部提供"对话历史"按钮，存在历史对话时按钮右上角显示紫色小圆点指示器
- 点击进入历史视图，顶部显示"对话历史"标题 + 对话总数 Badge + 返回按钮 + 新建对话按钮
- 对话列表按最后更新时间倒序排列
- 每条记录展示：对话图标、标题、最后消息预览（前 40 字符）、相对时间、消息数
- 当前活跃对话高亮显示
- 提供搜索框，支持按标题和消息内容模糊搜索
- 每条记录 hover 时显示删除按钮
- 历史为空时显示空状态和新建对话按钮
- 搜索无结果时显示"未找到匹配的对话"

### 4.8 数据持久化

- 对话数据持久化到 localStorage，键名 `oms-ai-conversations`
- 页面加载时从 localStorage 恢复历史对话（Date 字段从字符串重建）
- 对话数据变化时自动同步到 localStorage
- localStorage 读取失败时降级为空对话列表
- localStorage 写入失败时静默忽略（如容量超限）

### 4.9 对话内图表展示

当用户提出数据分析类问题时，AI 回复中应包含可视化图表，直接在对话气泡内渲染，无需跳转到仪表板页面。

#### 支持的图表类型

| 图表类型 | 适用场景 | 示例问题 |
|----------|----------|----------|
| 柱状图（Bar） | 分类对比、状态分布 | "各状态订单有多少""各仓库库存对比" |
| 折线图（Line） | 趋势变化、时间序列 | "最近 7 天订单趋势""本月销售走势" |
| 饼图（Pie） | 占比分布 | "订单来源占比""异常类型分布" |
| 数值卡片（Metric） | 单一指标展示 | "今天订单量""当前异常率" |
| 表格（Table） | 多维度明细数据 | "各仓库库存明细""Top 10 SKU 销量" |

#### 图表数据格式

AI 回复中使用特定语法标记图表数据，前端解析后渲染为对应组件：

```
:::chart
{
  "type": "bar",
  "title": "各状态订单数量",
  "data": [
    { "label": "待处理", "value": 45 },
    { "label": "已分仓", "value": 120 },
    { "label": "已发货", "value": 380 },
    { "label": "异常", "value": 12 }
  ]
}
:::
```

#### 图表展示规则

- 图表宽度自适应对话气泡宽度（最大约 320px）
- 图表高度固定 200px（数值卡片 80px）
- 配色遵循 Design System：主色 Primary、语义色（Success/Warning/Danger）
- 支持暗色模式
- 图表下方可附带文字解读
- 一条 AI 回复中最多包含 2 个图表，避免信息过载
- 图表不可交互（不支持 hover tooltip、点击钻取），如需深入分析引导用户前往对应页面

#### 触发条件

以下场景 AI 应在回复中包含图表：

- 用户问到"多少""对比""趋势""分布""占比"等数据分析关键词
- 工具返回的数据包含可聚合的数值字段（如各状态订单数、各仓库库存量）
- 用户明确要求"给我看个图""可视化一下"

## 5. Agent 架构

### 5.1 整体架构

AI 助手采用 Agent + Skill 架构。前端面板负责 UI 交互，后端 API 负责意图识别和 Skill 调度。

```
前端面板 → POST /api/ai/chat → Agent（LLM + System Prompt）
                                    ↓ 识别意图
                                    ↓ 选择 Skill / Tool
                                    ↓ 调用工具获取真实数据
                                    ↓ 用业务语言格式化结果
                                    ← 返回结构化回复
```

- System Prompt 从 `.kiro/skills/` 目录动态加载，包含 Agent 总控指令和各 Skill 的完整规则
- 前端不接触 System Prompt 内容
- 修改 Skill 文件后重启服务即生效，不需要改代码

| oms_query | 查询型 | 订单全景查询、状态、发运、仓库、分仓、规则、库存、Hold、时间线、集成中心 |

## 6. 分期规划

### 第一期 — 核心能力

核心目标：交付一个能真正解决业务问题的 AI 助手，覆盖运营人员日常最高频的查询、分析、计算场景。

#### 交互框架

| 功能 | 说明 | 状态 |

## 7. 不做什么（第一期）

- AI 不能直接执行任何写操作（不改数据，只给建议和链接）
- 不替代已有的 AI 异常处理页面（`/orders/exception-ai`），两者并存
- 不做移动端适配（当前只支持桌面端）
- 不做用户间的对话共享
- 不做对话导出功能
- 反馈按钮当前仅为 UI 展示，不上报数据
- 图表不支持交互（不支持 hover tooltip、点击钻取）
- 不做流式回复（第二期）
- 不做运费/时效/成本计算（第二期）

## 8. 竞品参考

### Shopify Sidekick

- 入口：后台右上角图标，点击打开侧边面板
- 定位：从"问答助手"演进为"AI 运营搭档"，能查数据、执行操作、构建自动化
- 关键特性：Pulse（主动洞察）、操作执行、应用生成、上下文感知

### 我们的差异

- 面向 B2B OMS 场景，不是 DTC 电商
- 核心价值在异常处理和供应链管理（含智能分仓和装箱计算），不是营销和内容生成
- 已有成熟的异常诊断管线和装箱引擎可以复用
- Agent + Skill 架构，新能力通过添加 Skill 文件扩展，不需要改代码
