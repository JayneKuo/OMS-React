# 产品需求文档：OMS AI 助手

## 1. 背景

OMS 系统功能模块多（订单、采购、库存、物流、退货、客户、自动化等），运营人员日常工作需要在多个页面间频繁切换。新员工上手周期长，老员工也经常需要多步操作才能完成一个简单任务（比如查一个订单的异常原因，需要先进订单列表 → 找到订单 → 进详情 → 看异常标签 → 再去事件管理看日志）。

同时，业务数据散落在各个模块中，运营主管想看一个跨模块的数据汇总（比如"本月异常率和上月对比"）往往需要自己导出数据手动分析。

Shopify 在 2024-2026 年推出并持续迭代的 Sidekick AI 助手验证了一个方向：在后台管理系统中嵌入一个自然语言交互的 AI 助手，可以显著降低操作门槛、提升运营效率。Sidekick 从最初的简单问答，演进到能执行操作、主动推送洞察、构建自动化工作流，成为商家的"AI 运营搭档"。

我们希望在 OMS 中引入类似的 AI 助手能力。

## 2. 目标

### 业务目标

- 降低运营人员的操作成本，减少完成常见任务所需的步骤和时间
- 降低新员工的上手门槛，通过 AI 引导快速熟悉系统
- 提升异常发现和处理的时效性，通过主动洞察让问题更早被发现

### 衡量指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 任务效率提升 | 常见查询任务耗时降低 50% | 对比有/无 AI 助手时完成同一任务的平均时间 |
| 用户采纳率 | 上线 1 个月后 ≥ 30% 日活用户使用 | 埋点统计面板打开次数 / DAU |
| 对话深度 | 平均 ≥ 3 轮/次 | 统计每次对话的消息轮次 |
| 新人上手周期 | 缩短 30% | 对比新员工独立完成首个任务的时间 |

## 3. 用户角色与场景

### 3.1 用户角色

| 角色 | 日常工作 | 痛点 |
|------|----------|------|
| 运营专员 | 处理订单异常、退货、发货跟踪 | 异常处理流程长，需要跨多个页面操作；重复性查询多 |
| 运营主管 | 监控业务指标、审批、团队管理 | 数据散落各模块，缺少跨模块汇总视角；异常发现滞后 |
| 新入职员工 | 学习系统、处理简单任务 | 不知道功能在哪、不清楚操作流程、怕操作错误 |
| 商户管理员 | 管理产品、库存、定价 | 产品和库存操作步骤多，批量操作不直观 |

### 3.2 核心场景

**场景 A：快速查数据**

运营主管每天早上想看昨天的业务概况。现在需要打开仪表板、切换时间范围、逐个看各项指标。希望能直接问一句"昨天的订单量和异常率是多少"就拿到答案。

**场景 B：异常处理**

运营专员收到异常通知，需要查原因。现在要进订单详情 → 看异常标签 → 去事件管理查日志 → 判断根因 → 决定处理方式。希望能输入订单号，AI 直接告诉根因和建议操作。

**场景 C：找功能入口**

新员工想处理一个退货，但不知道退货管理在哪。现在要问同事或者自己在菜单里找。希望能问 AI"退货管理在哪"，直接给一个链接点过去。

**场景 D：主动发现问题**

运营主管没有主动去看数据，但今天的异常率其实比昨天高了很多。现在这个问题要等到有人主动去看报表才会被发现。希望 AI 能主动提醒"今天异常率上升了 23%，要不要看看？"

**场景 E：操作指引**

商户管理员想创建一个新产品，但不确定流程。现在要看文档或问同事。希望能问 AI"怎么创建产品"，AI 给出步骤说明和对应页面的链接。

**场景 F：自动化需求**

运营主管想设置一个规则：库存低于安全水位时自动通知采购。现在需要自己去自动化模块研究怎么配。希望能用自然语言描述需求，AI 给出配置指引。

## 4. 功能描述

### 4.1 AI 助手入口

用户在系统的任意页面都能看到 AI 助手的入口，一键唤起。助手面板打开后不遮挡当前页面内容，用户可以一边看页面一边和 AI 对话。关闭后不影响当前工作，再次打开时之前的对话还在。

### 4.2 对话能力

用户用自然语言提问，AI 理解意图后给出回答。回答不只是纯文本，还能包含可以直接点击跳转的页面链接，减少用户自己找页面的步骤。

支持多轮对话，用户可以追问。支持同时维护多个对话（比如一个在聊订单异常，另一个在问库存），可以随时切换。

### 4.3 AI 能覆盖的能力域

| 能力域 | 用户能问什么 | AI 能做什么 |
|--------|-------------|------------|
| 数据与分析 | "本月销售额多少""和上月对比怎么样""帮我导出订单数据" | 返回数据摘要，提供报表页面链接 |
| 产品与库存 | "怎么创建产品""哪些商品快缺货了""怎么管理产品集合" | 操作步骤指引，提供对应页面链接 |
| 客户管理 | "帮我找高价值客户""某个客户的订单历史" | 查询指引，提供客户管理页面链接 |
| 定价与促销 | "有什么定价策略建议""最近促销效果怎么样" | 策略建议和分析指引 |
| 自动化 | "帮我创建一个低库存通知规则""有什么推荐的自动化" | 配置指引，提供自动化页面链接 |
| 导航与引导 | "退货管理在哪""怎么配置店铺设置""最近有什么新功能" | 直接给出页面链接，操作指引 |
| 订单与异常 | "SO12345 什么状态""今天有多少异常订单" | 订单查询，异常诊断指引 |

### 4.4 主动洞察

AI 不只是被动回答问题，还能主动分析业务数据，发现值得关注的变化并推送给用户。比如：

- 异常率突然上升
- 某个商品销量异常增长或下降
- 库存即将低于安全水位
- 某个时间段的退货率偏高

用户看到洞察后可以一键深入了解详情。不感兴趣的洞察可以关闭。

### 4.5 对话历史

用户的所有对话都会保留，可以随时回看之前的问答。支持搜索历史对话。可以删除不需要的对话。

## 5. 分期规划

### 第一期（MVP）— 本期交付

核心目标：跑通 AI 助手的交互框架，验证用户是否愿意用这种方式和系统交互。

| 功能 | 说明 |
|------|------|
| 助手入口与面板 | 全局可用，嵌入页面右侧，不遮挡内容 |
| 自然语言对话 | 多轮对话，多对话管理 |
| 快捷建议 | 按能力域分类的预设提问，降低使用门槛 |
| 结构化回复 | 文本 + 可点击页面链接 |
| 对话历史 | 保留、搜索、删除 |
| 主动洞察 | 展示预设的洞察卡片（静态数据） |
| 模拟回复 | 基于关键词匹配的预设回复，不依赖后端 AI |

本期的 AI 回复是预设的模拟内容，目的是先把交互框架搭好，收集用户反馈，验证产品方向。

### 第二期 — 接入真实 AI

| 功能 | 说明 |
|------|------|
| 真实 AI 回复 | 接入 LLM，理解用户意图，生成个性化回复 |
| 异常诊断集成 | 复用已有的异常诊断管线，输入订单号直接出诊断结果 |
| 真实数据洞察 | Pulse 接入仪表板数据 API，实时计算并推送洞察 |
| 流式回复 | 打字机效果，提升响应体验 |

### 第三期 — 操作执行

| 功能 | 说明 |
|------|------|
| 操作执行 | AI 不只给建议，还能直接执行操作（重开订单、创建折扣等） |
| 上下文感知 | 根据用户当前所在页面自动理解上下文 |
| 语音输入 | 支持语音交互 |

## 6. 不做什么

- 本期不接入真实 AI/LLM，只做模拟回复
- 本期 AI 不能直接执行任何写操作（不改数据，只给建议和链接）
- 不替代已有的 AI 异常处理页面（`/orders/exception-ai`），两者并存
- 不做移动端适配（本期只支持桌面端）
- 不做用户间的对话共享
- 不做对话导出功能

## 7. 竞品参考

### Shopify Sidekick

- 入口：后台右上角图标，点击打开侧边面板
- 定位：从"问答助手"演进为"AI 运营搭档"，能查数据、执行操作、构建自动化
- 关键特性：
  - Pulse（主动洞察）：持续分析店铺数据，主动推送建议
  - 操作执行：一句话创建折扣码、配置工作流
  - 应用生成：用自然语言描述需求，自动生成 Shopify 应用
  - 上下文感知：在主题编辑器中可以直接修改设计

### 我们的差异

- 面向 B2B OMS 场景，不是 DTC 电商
- 核心价值在异常处理和供应链管理，不是营销和内容生成
- 已有成熟的异常诊断管线可以复用
- 第一期先验证交互框架，不急于上 AI 能力

## 8. 开放问题

| 问题 | 状态 | 备注 |
|------|------|------|
| 第二期用什么 LLM？自建还是调第三方？ | 待定 | 需要评估成本和数据安全 |
| 对话数据是否需要上报到服务端？ | 待定 | 当前存 localStorage，后续可能需要服务端存储以支持跨设备 |
| 是否需要限制对话历史条数？ | 待定 | localStorage 有容量限制，大量对话可能超出 |
| Pulse 洞察的数据源和计算逻辑？ | 待定 | 第一期用静态数据，第二期需要定义具体的指标和阈值 |
| 是否需要管理员配置 AI 的回复内容？ | 待定 | 比如自定义快捷建议、调整能力域 |
